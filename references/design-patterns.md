# 设计模式在 Tiecode 中的应用

## 1. 单例模式

### 1.1 使用 @禁止创建对象 + @静态

```t
@禁止创建对象
类 单例测试 : 父类
	@隐藏
	@静态
	变量 测试 : 测试类?
	
	@静态
	方法 单例测试() : 测试
		如果 测试 == 空
			code #测试 = new #<测试类>();
		结束 如果
		返回 测试
	结束 方法
结束 类
```

调用方式：
```t
变量 实例 : 测试 = 单例测试.单例测试()
```

规则：
- `@禁止创建对象` 禁止外部实例化
- `@静态` 标记的成员通过类名直接访问
- `变量 测试 : 测试类?` 只声明引用，不自动创建
- `@隐藏` 隐藏内部实现细节
- `code #测试 = new #<测试类>()` 在 code 中创建实例并赋值
- 通过静态方法控制唯一实例的获取

## 2. 观察者模式（事件驱动）

### 2.1 使用定义事件

```t
类 数据仓库
	变量 缓存数据 : 文本
	
	方法 刷新数据()
		变量 结果 : 文本 = 从网络获取()
		如果 结果 != 空 则
			缓存数据 = 结果
			数据已更新(结果)
		否则
			数据加载失败("网络请求失败")
		结束 如果
	结束 方法
	
	定义事件 数据已更新(新数据 : 文本)
	定义事件 数据加载失败(错误信息 : 文本)
结束 类
```

订阅者实现：
```t
类 主页面 : 窗口
	变量 仓库 : 数据仓库
	
	事件 主页面:创建完毕()
		仓库.刷新数据()
	结束 事件
	
	事件 仓库:数据已更新(新数据 : 文本)
		更新界面(新数据)
	结束 事件
	
	事件 仓库:数据加载失败(错误信息 : 文本)
		显示提示(错误信息)
	结束 事件
结束 类
```

规则：
- `定义事件` 放在类成员末尾，在调用它的方法之后
- 订阅者通过 `事件 对象:事件名(...)` 实现响应
- 适合 UI 更新、消息通知、状态同步等场景

## 3. 适配器模式

### 3.1 列表适配器

```t
类 列表适配器
	变量 数据源 : 文本[]
	
	事件 列表适配器:关联项目数量() : 整数
		返回 取数组长度(数据源)
	结束 事件
	
	事件 列表适配器:关联视图(项目类型 : 整数) : 组件容器
		变量 布局 : 组件容器
		// 加载列表项布局
		返回 布局
	结束 事件
	
	事件 列表适配器:关联视图数据(项目类型 : 整数, 容器 : 组件容器, 索引 : 整数)
		// 绑定数据到视图
	结束 事件
	
	定义事件 关联项目数量() : 整数
	定义事件 关联视图(项目类型 : 整数) : 组件容器
	定义事件 关联视图数据(项目类型 : 整数, 容器 : 组件容器, 索引 : 整数)
结束 类
```

规则：
- 适配器通过 `定义事件` 暴露数据绑定点
- `定义事件` 放在类成员末尾
- 调用方实现事件来提供数据和视图
- 适合列表、网格等数据展示场景

## 4. 模板方法模式

### 4.1 使用 @虚拟方法

```t
类 基础页面 : 窗口
	事件 基础页面:创建完毕()
		初始化界面()
		加载数据()
	结束 事件
	
	@虚拟方法
	方法 初始化界面()
	结束 方法
	
	@虚拟方法
	方法 加载数据()
	结束 方法
结束 类

类 商品页面 : 基础页面
	事件 商品页面:初始化界面()
		// 具体实现
	结束 事件
	
	事件 商品页面:加载数据()
		// 具体实现
	结束 事件
结束 类
```

规则：
- 父类使用 `@虚拟方法` 声明可重写点
- 子类通过事件实现具体逻辑
- 适合页面框架、业务流程模板等场景
