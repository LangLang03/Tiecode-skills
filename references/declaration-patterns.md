# 声明模式库（类/方法/属性/事件）

## 1. 包、导入、映射
```t
类 启动信息
结束 类
```

规则：
- 默认不写 `包名`。
- 未使用内嵌 Java 时，不写 `@导入Java`。
- 仅在桥接 Java 类型时才添加 `@导入Java` 与 `@指代类`。

## 2. 类定义模板

基础类：
```t
类 数据对象
结束 类
```

继承：
```t
类 文本类 : 字符串
结束 类
```

泛型：
```t
类 集合模板类<模板类型1>
结束 类
```

禁止创建对象：
```t
@禁止创建对象
类 工具类
结束 类
```

## 3. 方法定义模板

普通方法：
```t
方法 取名称() : 文本
	code return #this.getName();
结束 方法
```

默认参数：
```t
方法 截取(开始 : 整数 = 0, 结束 : 整数 = -1) : 文本
结束 方法
```

构造语义：
```t
@运算符重载
方法 =(路径 : 文本)
	@code
	// init
	@end
结束 方法
```

## 4. 属性模板

只读属性：
```t
属性读 长度() : 整数
	code return #this.length();
结束 属性
```

只写属性：
```t
属性写 标题(标题 : 文本)
	code getView().setTitle(#标题);
结束 属性
```

读写成对属性：
```t
属性写 选中(是否选中 : 逻辑型)
	code getView().setChecked(#是否选中);
结束 属性

属性读 选中() : 逻辑型
	code return getView().isChecked();
结束 属性
```

属性访问语义（强制）：
- `属性读 XXX() : 类型` 定义后，允许使用 `对象.XXX` 读取值。
- `属性写 XXX(值 : 类型)` 定义后，允许使用 `对象.XXX = 值` 赋值。
- `属性读/属性写` 可以同名，建议同名配对以保持直观。

示例（同名读写）：
```t
属性写 宽度(值 : 整数)
	code #this.setWidth(#值);
结束 属性

属性读 宽度() : 整数
	code return #this.getWidth();
结束 属性
```

对应调用：
```t
变量 组件1 : 可视化组件
组件1.宽度 = -1
变量 当前宽度 : 整数 = 组件1.宽度
```

布局属性注解方法（可作为布局字段）：
```t
@布局属性
方法 位于某组件之下(欲设置组件 : 可视化组件, 目标组件ID 为 整数)
	@code
	// 布局规则实现
	@end
结束 方法

@布局属性
方法 位于布局中间(欲设置组件 : 可视化组件, 是否居中 为 逻辑型)
	@code
	// 布局规则实现
	@end
结束 方法
```

规则：
- 被 `@布局属性` 标记的方法可在 `@布局配置` 中作为布局字段使用。
- 生成布局字段前，必须在源码或索引中检索到该方法与注解（例如 `位于某组件之下`）。
- `@布局属性` 方法只允许两个参数：第一个固定为 `欲设置组件 : 可视化组件`，第二个为布局参数值。
- 在 `@布局配置` 中调用时必须写成 `@方法名=值`，例如 `@位于某组件之下=1`。
- 布局参数值不只支持数字：可根据第二参数类型接收文本/逻辑型/数字等值（例如 `@位于布局中间=真`）。
- 普通属性键同样支持多类型值（例如 `内容="猜硬币游戏"`、`字体大小=24`），前提是对应属性写入存在且类型匹配。

布局调用示例：
```t
@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=2,内容="提示文本",字体大小=18,@位于某组件之下=1,@位于布局中间=真]])
变量 分数文本 : 文本框
```

## 5. 变量与常量

常量：
```t
@静态
常量 压缩等级_默认 : 整数 = -1
```

变量：
```t
变量 索引 : 整数 = 0
变量 缓冲 : 字节[1024]
```

对象变量创建规则（强制）：
- 对象变量使用类型声明时会自动创建对象：`变量 对象名 : 类型`。
- 禁止生成 `变量 对象名 : 类型 = 创建 类型()`。
- `创建` 关键字属于内部机制，业务代码中不应直接使用。

错误示例：
```t
变量 计算器 : 数学表达式计算器 = 创建 数学表达式计算器()
```

正确示例：
```t
变量 计算器 : 数学表达式计算器
```

函数签名匹配规则（强制）：
- 函数调用参数数量与类型必须与已定义签名一致，不可自行扩展重载。
- 函数调用形态必须匹配方法类别：实例方法必须写 `实例.方法(...)`，静态方法必须写 `类型.方法(...)`。
- 示例：文本转小数优先使用实例方法 `文本值.到单精度小数()`。
- 禁止示例：`到小数(文本值)`（若库中仅存在无参 `到小数()`，则该调用非法）。
- 禁止示例：`到文本(变量名)`（把实例方法误写为全局调用）；正确写法：`变量名.到文本()`。

## 6. 事件模板

声明事件：
```t
定义事件 数据已加载(条数 : 整数)
```

带返回值事件：
```t
定义事件 比较对象(对象1 : 模板类型1, 对象2 : 模板类型1) : 整数
```

实现事件：
```t
事件 适配器对象:加载布局(索引 : 整数, 项目布局 : 组件容器) : 组件容器
	返回 加载布局(索引, 项目布局)
结束 事件
```

## 7. 返回规则
- 有返回值时优先写显式返回类型。
- `返回` 语句写在 结绳层；`code return ...` 写在内嵌 Java 层。
- 不要混用 `返回` 与 `code return` 指向同一逻辑路径。
