# 声明模式库（类/方法/属性/事件）

## 1. 包、导入、映射
```t
类 启动信息
结束 类
```

规则：
- 默认不写 `包名`。
- 未使用内嵌 Java 时，不写 `@导入Java`。
- 仅在桥接 Java 类型时才添加 `@导入Java` 与 `@指代类`。
- `@导入Java` 为类级独立声明，不跨类自动继承；同文件多类时按类分别声明所需导入。

## 2. 类定义模板

基础类：
```t
类 数据对象
结束 类
```

继承：
```t
类 文本类 : 字符串
结束 类
```

泛型：
```t
类 集合模板类<模板类型1>
结束 类
```

禁止创建对象：
```t
@禁止创建对象
类 工具类
结束 类
```

`@禁止创建对象` 使用规则（强制）：
- 先查看类定义，确认可用成员（通常为 `@静态` 方法/常量）。
- 禁止任何实例化方式：`变量 名称 : 工具类`（自动创建）与 `变量 名称 : 工具类 = 创建 工具类()`（显式创建）都不允许。
- 调用时优先使用类型接收者：`工具类.方法(...)`。

类结构规则（强制）：
- 一个 `.t` 文件允许定义多个顶层类（`类 ... 结束 类` 可重复出现）。
- 不支持内部类/嵌套类语法；不得在类体内再声明 `类 ...`。
- `结束 类` 仅闭合当前最近未闭合的类声明，不可跨层复用。
- 标识符（类/方法/变量/参数/常量/事件）不得使用语法关键字命名。

可变清单参数承载模板（强制）：
- 使用 `@附加可变清单` 时，应新建独立类承载参数映射。
- 承载类仅保留一个空方法，不写运行时逻辑。
- 方法参数与模板占位符 `${参数名}` 对齐。

示例：
```t
类 模板参数映射
	@静态
	@编译时处理参数
	@附加可变清单([[...${输入法类名}...${输入法名称}...${输入法配置}...]])
	方法 XXX(输入法类名:文本,输入法名称:文本,输入法配置:文本)
	结束 方法
结束 类
```

## 3. 方法定义模板

普通方法：
```t
方法 取名称() : 文本
	code return #this.getName();
结束 方法
```

默认参数：
```t
方法 截取(开始 : 整数 = 0, 结束 : 整数 = -1) : 文本
结束 方法
```

构造语义：
```t
@运算符重载
方法 =(路径 : 文本)
	@code
	// init
	@end
结束 方法
```

## 4. 属性模板

只读属性：
```t
属性读 长度() : 整数
	code return #this.length();
结束 属性
```

只写属性：
```t
属性写 标题(标题 : 文本)
	code getView().setTitle(#标题);
结束 属性
```

读写成对属性：
```t
属性写 选中(是否选中 : 逻辑型)
	code getView().setChecked(#是否选中);
结束 属性

属性读 选中() : 逻辑型
	code return getView().isChecked();
结束 属性
```

属性访问语义（强制）：
- `属性读 XXX() : 类型` 定义后，允许使用 `对象.XXX` 读取值。
- `属性写 XXX(值 : 类型)` 定义后，允许使用 `对象.XXX = 值` 赋值。
- `属性读/属性写` 可以同名，建议同名配对以保持直观。

示例（同名读写）：
```t
属性写 宽度(值 : 整数)
	code #this.setWidth(#值);
结束 属性

属性读 宽度() : 整数
	code return #this.getWidth();
结束 属性
```

对应调用：
```t
变量 组件1 : 可视化组件
组件1.宽度 = -1
变量 当前宽度 : 整数 = 组件1.宽度
```

布局属性注解方法（可作为布局字段）：
```t
@布局属性
方法 位于某组件之下(欲设置组件 : 可视化组件, 目标组件ID 为 整数)
	@code
	// 布局规则实现
	@end
结束 方法

@布局属性
方法 位于布局中间(欲设置组件 : 可视化组件, 是否居中 为 逻辑型)
	@code
	// 布局规则实现
	@end
结束 方法
```

规则：
- 被 `@布局属性` 标记的方法可在 `@布局配置` 中作为布局字段使用。
- 生成布局字段前，必须在源码或索引中检索到该方法与注解（例如 `位于某组件之下`）。
- `@布局属性` 方法只允许两个参数：第一个固定为 `欲设置组件 : 可视化组件`，第二个为布局参数值。
- 在 `@布局配置` 中调用时必须写成 `@方法名=值`，例如 `@位于某组件之下=1`。
- `@位于某组件之下` / `@位于某组件之上` 仅表示 2D 平面相对位置（上下关系），不改变组件层级（父子关系）与显示层次（z 顺序）。
- 布局参数值不只支持数字：可根据第二参数类型接收文本/逻辑型/数字等值（例如 `@位于布局中间=真`）。
- 普通属性键同样支持多类型值（例如 `内容="猜硬币游戏"`、`字体大小=24`），前提是对应属性写入存在且类型匹配。

布局调用示例：
```t
@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=2,内容="提示文本",字体大小=18,@位于某组件之下=1,@位于布局中间=真]])
变量 分数文本 : 文本框
```

## 5. 变量与常量

常量：
```t
@静态
常量 压缩等级_默认 : 整数 = -1
```

变量：
```t
变量 索引 : 整数 = 0
变量 文本列表 : 文本[]
变量 表格 : 文本[][]
变量 立方体 : 文本[][][]
变量 数字集 : 整数[] = {1,2,3}
变量 棋盘 : 整数[][] = {{1,2},{3,4}}
```

对象变量创建规则（强制）：
- 对象变量使用类型声明时会自动创建对象：`变量 对象名 : 类型`。
- 禁止生成 `变量 对象名 : 类型 = 创建 类型()`。
- `创建` 关键字属于内部机制，业务代码中不应直接使用。
- `变量 对象名 : 类型?` 表示只声明引用，不自动创建对象（用于延迟赋值或禁止创建对象场景）。
- 数组支持花括号初始化：`变量 名称 : 类型[] = {值1,值2,值3}`；多维数组使用嵌套花括号。

错误示例：
```t
变量 计算器 : 数学表达式计算器 = 创建 数学表达式计算器()
```

正确示例：
```t
变量 计算器 : 数学表达式计算器
```

只声明示例（不自动创建）：
```t
变量 计算器 : 数学表达式计算器?
```

`@禁止创建对象` 场景示例：
```t
@禁止创建对象
类 工具类
	@静态
	方法 到文本(值 : 整数) : 文本
	结束 方法
结束 类

变量 工具 : 工具类?  ' 仅声明引用，不触发创建
变量 文本值 : 文本 = 工具类.到文本(1)
```

函数签名匹配规则（强制）：
- 函数调用参数数量与类型必须与已定义签名一致，不可自行扩展重载。
- 函数调用形态必须匹配方法类别：实例方法必须写 `实例.方法(...)`，静态方法必须写 `类型.方法(...)`。
- 示例：文本转单精小数优先使用实例方法 `文本值.到单精小数()`（或项目中已定义的等价名称）。
- 禁止示例：`到小数(文本值)`（若库中仅存在无参 `到小数()`，则该调用非法）。
- 禁止示例：`到文本(变量名)`（把实例方法误写为全局调用）；正确写法：`变量名.到文本()`。

`小数(double)` 与 `单精小数(float)` 转换规则（强制）：
- `小数 -> 单精小数` 在结绳层不支持直接转换。
- 仅在“必须转换”的地方，允许两种方式：`小数值.到文本().到单精小数()`（或项目中已定义的等价方法名）/ `@code` 代码块转换。
- 所有转换方法必须可检索、已定义；禁止发明不存在的转换函数。

示例：
```t
变量 d源 : 小数 = 123.456789
变量 f源 : 单精度小数 = d源.到文本().到单精小数()  ' 仅在必须转换时使用；若项目方法名不同，使用已定义等价方法
```

## 6. 事件模板

声明事件：
```t
定义事件 数据已加载(条数 : 整数)
```

带返回值事件：
```t
定义事件 比较对象(对象1 : 模板类型1, 对象2 : 模板类型1) : 整数
```

实现事件：
```t
事件 适配器对象:加载布局(索引 : 整数, 项目布局 : 组件容器) : 组件容器
	返回 加载布局(索引, 项目布局)
结束 事件
```

## 7. 返回规则
- 有返回值时优先写显式返回类型。
- `返回` 语句写在 结绳层；`code return ...` 写在内嵌 Java 层。
- 不要混用 `返回` 与 `code return` 指向同一逻辑路径。
