# 常见业务场景实现

## 1. 列表数据场景

### 1.1 自定义列表框实现

```t
类 启动窗口 : 窗口
	@布局配置([[根布局=真,宽度=-1,高度=-1,纵向布局=真]])
	变量 线性布局1 : 线性布局
	@布局配置([[父布局=线性布局1]])
	变量 自定义列表框1 : 自定义列表框
	
	变量 列表数据 : 文本[] = {
		"项目一", "数据2"
	}
	
	事件 启动窗口:创建完毕()
		订阅事件()
		自定义列表框1.更新项目(取数组长度(列表数据))
	结束 事件
	
	// 列表加载布局事件
	事件 自定义列表框1:加载布局(索引 : 整数, 项目布局 : 组件容器) : 组件容器
		// 先定义一个自定义布局参考变量
		变量 布局 : 我的列表布局?
		// 判断当前正在加载的列表项目布局是否为空
		如果 项目布局 == 空
			// 如果为空，则创建新布局
			布局 = 创建 我的列表布局
		否则
			// 如果不为空，则使用布局缓存
			布局 = 项目布局
		结束 如果
		// 设置自定义布局的文本框内容
		布局.文本框1.内容 = 列表数据[索引]
		// 最后返回布局
		返回(布局)
	结束 事件
	
	// 列表项目被单击事件
	事件 自定义列表框1:项目被单击(被单击组件 : 可视化组件, 索引 : 整数)
		弹出提示(列表数据[索引])
	结束 事件
结束 类

/*
列表自定义布局
*/
类 我的列表布局 : 组件容器
	@布局配置([[根布局=真,宽度=-1,高度=200,对齐方式=8388627]])
	变量 线性布局1 : 线性布局
	@布局配置([[父布局=线性布局1,@权重=1.0]])
	变量 文本框1 : 文本框
结束 类
```

规则：
- 数据源使用数组存储
- `变量 布局 : 我的列表布局?` 只声明引用，不自动创建
- `布局 = 创建 我的列表布局` 在需要时创建布局
- 布局缓存复用：判断 `项目布局 == 空` 决定是否创建新布局
- `更新项目(数量)` 方法更新列表显示
- 自定义布局类继承 `组件容器`
- 注释使用 `//` 单行或 `/* */` 多行

## 2. 网络请求场景

### 2.1 异步网络请求

```t
@异步方法
方法 加载用户数据(用户ID : 文本)
	变量 响应 : 文本 = 等待 网络.取网页源码_同步("https://api.example.com/user/" + 用户ID)
	如果 响应 != 空 则
		解析并更新界面(响应)
	否则
		显示错误("网络请求失败")
	结束 如果
结束 方法
```

规则：
- `@异步方法` 标记异步执行的方法
- `等待` 关键字等待异步操作完成
- 适合网络请求、文件读写等耗时操作

### 2.2 带回调的网络请求

```t
类 网络请求器
	方法 发送GET请求(网址 : 文本)
		提交到新线程运行()
			变量 响应 : 文本 = 网络.取网页源码_同步(网址)
			提交到主线程运行(本对象)
				如果 响应 != 空 则
					请求成功(响应)
				否则
					请求失败("请求失败")
				结束 如果
			结束提交到主线程()
		结束提交到新线程()
	结束 方法
	
	定义事件 请求成功(响应数据 : 文本)
	定义事件 请求失败(错误信息 : 文本)
结束 类
```

规则：
- `提交到新线程运行()` 在后台线程执行
- `提交到主线程运行(窗口对象)` 切回主线程更新 UI
- 网络请求必须在后台线程执行
- `定义事件` 放在类成员末尾

## 3. 页面导航场景

### 3.1 页面跳转传参

```t
类 登录页面 : 窗口
	变量 用户名输入框 : 编辑框
	变量 登录按钮 : 按钮
	
	事件 登录按钮:被单击()
		变量 用户名 : 文本 = 用户名输入框.内容
		变量 参数 : 启动信息
		参数.置文本("username", 用户名)
		切换窗口(主页面对象, 参数)
	结束 事件
结束 类

类 主页面 : 窗口
	事件 主页面:创建完毕(参数 : 启动信息)
		变量 用户名 : 文本 = 参数.取文本("username", "")
		显示欢迎信息(用户名)
	结束 事件
结束 类
```

规则：
- 使用 `启动信息` 对象传递参数
- `切换窗口(目标窗口, 参数)` 跳转并传参
- 目标页面在 `创建完毕` 事件中接收参数
