# 常见错误与修复规则

## 0. 违反最高优先级 Agent 规则
错误：在支持 Agent 且用户未明确禁止时，未使用 Agent 覆盖“查看文档/理解项目/审查生成代码”三阶段。  
修复：立即补执行 Agent 三阶段流程，并根据 Agent 审查结果回改代码后再继续后续规则检查。

## 1. 块未闭合
症状：编译器在文件末尾报语法错误。  
修复：按最近开启块补齐 `结束 方法/类/属性/事件/循环/如果/假如`。

## 2. `@code` 未闭合
症状：后续 结绳 被当作 Java 文本。  
修复：补 `@end`，并检查嵌套匿名类括号。

## 3. 参数缺类型或分隔符
错误：`方法 查找(关键词, 起始 : 整数)`  
修复：`关键词 : 文本`（每个参数都要类型）。

## 4. 事件声明和实现混用
错误：用 `定义事件` 写方法体。  
修复：
- 仅声明：`定义事件 ...`
- 有方法体：`事件 ...` + `结束 事件`

## 5. 结绳/Java return 混用
错误：在 `@code` 中写 `返回`。  
修复：在 Java 块用 `return`，结绳层用 `返回`。

## 6. 漏加 `#` 宏前缀
错误：`setText(标题)`  
修复：`setText(#标题)`。

## 7. 运算符方法缺注解
错误：定义 `方法 [](...)` 但未标 `@运算符重载`。  
修复：在该方法前加 `@运算符重载`。

## 8. 注解位置错误
错误：把 `@附加权限` 写在方法体中。  
修复：移动到方法声明前。

## 9. 风格漂移
错误：同文件既用 `:` 又用 `为` 返回类型。  
修复：按文件主风格统一一种。

## 10. 过度 API 化
错误：大段罗列 API 而无语法模板。  
修复：按“语法模板 > 示例 > API 索引”顺序组织。

## 11. 页面继承错误
错误：`类 测试页面 : 组件容器`。  
修复：页面类改为 `类 测试页面 : 窗口`。

## 12. 主窗口命名/继承错误
错误：主入口窗口不是 `类 启动窗口 : 窗口`。  
修复：主入口统一为 `类 启动窗口 : 窗口`。

## 13. 无必要 `包名`
错误：默认生成 `包名 结绳.安卓`。  
修复：默认移除 `包名`，仅在用户明确要求时保留。

## 14. 无必要 `@导入Java`
错误：未使用内嵌 Java 却导入 `@导入Java(...)`。  
修复：仅在 `code` 或 `@code...@end` 被实际使用时添加导入。

## 15. 构造器样板重复生成
错误：类中已存在构造器样板，却再次生成：
```t
@code
public #<测试页面>(android.content.Context context) {
	super(context);
}
@end
```
修复：先检查是否已定义；已定义则保留一份并删除重复项；缺失且必要时补一份。

## 16. 误改 `绳包/` 目录
错误：在未授权情况下修改 `绳包/` 代码。  
修复：回到 `源代码/` 完成功能；`绳包/` 视为依赖目录，不在其任何位置写代码。

## 17. 调用未检索函数
错误：直接调用函数但未在 `源代码/` 和 `绳包/` 中找到定义。  
修复：先检索定义；若未命中则改用已存在函数，或显式补充定义后再调用。

## 18. 假设未公开内置函数
错误：使用“系统自带但未公开”的函数名。  
修复：移除该调用；仅使用可检索、可证明存在的公开函数。

## 19. 属性读写未定义却直接访问
错误：未定义 `属性读/属性写` 就使用 `对象.属性` 或 `对象.属性 = 值`。  
修复：先补齐对应 `属性读` 或 `属性写`，或改为已存在成员调用。

## 20. 布局配置字段无属性支持
错误：`@布局配置([[父布局=根布局,宽度=-1,高度=-2]])` 中字段在目标组件上找不到对应属性读写支持。  
修复：先在 `源代码/` 与 `绳包/` 检索对应属性；若字段属于布局属性方法（如 `位于某组件之下`），继续检索是否有 `@布局属性` 注解方法支持；都不存在则替换为可用字段或补充可追踪定义后再使用。内置结构键 `父布局/根布局` 允许直接使用，不要求在基本库中找到定义。

## 21. 误判属性同名冲突
错误：将同名 `属性读/属性写` 视为冲突并删除其一。  
修复：保留同名读写对；仅在签名重复且语义冲突时重命名。

## 22. 布局属性方法被当作未知字段
错误：`@布局配置` 中使用 `位于某组件之下` 等字段时，误判为非法键。  
修复：检查该字段是否对应 `@布局属性` 注解方法；若存在则允许作为布局字段使用。

## 23. `@布局属性` 方法参数数量错误
错误：`@布局属性` 方法定义为 1 个或 3+ 个参数。  
修复：改为固定双参数：`(欲设置组件 : 可视化组件, 参数值 : 类型)`。

## 24. 布局属性方法键缺少 `@` 前缀
错误：`@布局配置([[...,位于某组件之下=1]])`。  
修复：改为 `@布局配置([[...,@位于某组件之下=1]])`。

## 25. 多组件布局声明中插入空行
错误：类定义后未空一行，或在连续的 `@布局配置` + `变量` 声明对之间插入空行（尤其 `@布局配置` 定义之间出现空行）导致解析报错。  
修复：类定义后保留一个空行，再连续声明布局对；`@布局配置` 定义之间不允许出现空行。布局声明块结束后是否空行不作限制。

错误示例（错在中间空行，不是错在结尾空行）：
```t
类 测试页面 : 窗口

	@布局配置([[根布局=真,宽度=-1,高度=-1]])
	变量 根布局 : 相对布局

	@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=1]])
	变量 标题文本 : 文本框
结束 类
```

正确示例：
```t
类 测试页面 : 窗口

	@布局配置([[根布局=真,宽度=-1,高度=-1]])
	变量 根布局 : 相对布局
	@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=1]])
	变量 标题文本 : 文本框
结束 类
```

## 26. 将布局值错误限制为数字
错误：认为 `@布局配置` 只支持数字值，拒绝 `文本/逻辑型`。  
修复：按目标属性或 `@布局属性` 方法参数类型赋值；例如 `内容="提示文本"`、`字体大小=18`、`@位于布局中间=真` 都可合法。

## 27. 布局值类型不匹配或字面量写法错误
错误：
- 文本值未加引号（如 `内容=提示文本`）
- 逻辑值写成字符串（如 `@位于布局中间="真"`）
- 数字值误写为文本（如 `字体大小="18"`）
修复：文本用引号、逻辑型用 `真/假`、数字保持数值字面量，并与目标签名类型一致。

## 28. 使用英文布局键名
错误：在 `@布局配置` 中写 `height=-2`、`width=-2` 等英文键。  
修复：统一改为中文规范键：`高度=-2`、`宽度=-2`。

## 29. UI 文本出现无意空格
错误：生成 `猜 正面`、`猜 反面`、`总次数:0 | 胜利:0 | 失败:0` 等带无意空格的文本。  
修复：默认移除无意空格，改为 `猜正面`、`猜反面`、`总次数:0|胜利:0|失败:0`；仅在用户明确要求时保留空格风格。

## 30. 文档检索范围错误（项目 skill / 全局 skill 混淆）
错误：已加载全局 skill 却去工作区 `skills/**` 搜文档，或已加载项目 skill 却读了其他 skill 文档。  
修复：
- 先判定 skill 模式（全局/项目）。
- 文档只从“当前已加载 skill 根目录”的 `references/` 读取。
- 除非用户明确要求，禁止跨 skill、跨工作区文档扫描。

## 31. 文档路径获取方式错误（未用脚本或未回退）
错误：
- Python 可用却未运行 `scrpits/list_doc_absolute_paths.py`，直接在工作区盲搜路径。
- Python 不可用时未回退到 `SKILL.md` 的 Mandatory Read Set 相对路径。
修复：
- Python 可用：先运行脚本，按输出绝对路径读取文档。
- Python 不可用：严格使用 Mandatory Read Set 原相对路径，不变通到其他工作区文档。

## 32. 未执行继承链分析（第0标识）
错误：未先分析“当前类 -> 父类 -> 目标参数类型”的继承关系，直接生成成员调用或参数传递。  
修复：在生成代码前强制完成继承链分析；该规则适用于所有类，不止窗口类。先判定可上转类型，再决定调用与传参。

## 33. 在窗口继承类中误用 `本对象.取安卓环境()`
错误：类继承 `窗口`（含间接继承）时写出 `本对象.取安卓环境()`。  
修复：此场景中 `本对象` 已是 `安卓环境(Context)`，直接传 `本对象`。`取安卓环境()` 仅用于当前对象不是 `安卓环境` 且确实需要取环境的场景。

## 34. 将 `父布局` 误判为必须在基本库定义
错误：因为未在基本库检索到 `父布局` 定义而判定语法错误。  
修复：`父布局/根布局` 属于布局系统内置结构键，可直接在 `@布局配置` 中使用；其他非内置键仍需通过属性读写或 `@布局属性` 方法证明可用。

## 35. 条件判断把 `=` 当作比较符
错误：`如果 硬币结果 = 0 则`。  
修复：条件比较使用 `==` / `!=`，`=` 仅用于赋值与默认值。正确写法：`如果 硬币结果 == 0 则`。

## 36. 可用且未禁用时未使用 Agent
错误：在支持 Agent 且用户未禁止的情况下，未使用 Agent 查看文档/理解项目/审查生成代码。  
修复：将 Agent 作为强制流程执行：生成前由 Agent 扫描文档并理解项目，生成后由 Agent 审查代码并按审查结果修复。

## 37. 布局颜色值误用十六进制
错误：在 `@布局配置` 中写 `背景颜色=0xfff5f5f5`、`背景颜色=0XFF2C3E50` 等十六进制字面量。  
修复：布局颜色统一使用十进制有符号整数，例如 `背景颜色=-657931`、`背景颜色=-13877680`。

## 38. 带颜色布局定义之间插入空行
错误：如下写法在两个 `@布局配置` 定义间插入空行：  
```t
@布局配置([[根布局=真,宽度=-1,高度=-1,背景颜色=-657931]])
变量 根布局 : 相对布局

@布局配置([[父布局=根布局,宽度=-1,高度=200,ID=1,背景颜色=-13877680,@位于布局顶部=真]])
变量 显示区域 : 相对布局
```
修复：删除中间空行，保持连续定义。

## 39. 将 Java SE API 误用于 Android
错误：在 Android 目标代码中直接导入/调用 Java SE 专属 API（如 `java.awt.*`、`javax.swing.*`、`java.applet.*`）。  
修复：先做 Android 可用性校验；仅使用 Android/AndroidX/依赖库中可用 API，或改写为项目已有可用实现。

## 40. 未校验 Java API 的 Android 可用性
错误：使用 `@导入Java` 或 `@code` 时，未验证 API 是否在 Android 运行时可用。  
修复：把“Android 可用性校验”作为前置步骤；无法证明可用时禁止生成该 API 调用。

## 41. 对象变量误用 `创建` 关键字
错误：`变量 计算器 : 数学表达式计算器 = 创建 数学表达式计算器()`。  
修复：对象变量直接使用类型声明自动创建，写成 `变量 计算器 : 数学表达式计算器`；`创建` 关键字仅内部使用，不在业务代码中生成。

## 42. 多分支误写为 `否则如果`
错误：写成 `否则如果 条件2 则`。  
修复：改为 `否则 条件2` 写法，并保持 `结束 如果` 闭合。

## 43. 条件循环缺少括号
错误：写成 `循环 当前位置 < 表达式文本.长度`。  
修复：改为 `循环(当前位置 < 表达式文本.长度)`。

## 44. 调用不存在签名的重载形式
错误：未校验函数签名就写 `到小数(数字文本)`。  
修复：按已定义签名调用，优先使用可检索实例方法（如 `数字文本.到单精度小数()`），或改为库中真实存在的调用形式。

## 45. 未区分静态方法与实例方法调用形态
错误：把实例方法写成全局调用或静态调用，例如 `到文本(变量名)`。  
修复：先确认方法类别再调用：实例方法使用 `变量名.到文本()`，静态方法使用 `类型名.方法(...)`；禁止无接收者的全局误调用。
