# 模板配方（高复用实例）

## 1. 主窗口页面模板（推荐默认）
```t
类 启动窗口 : 窗口
	@布局配置([[根布局=真,宽度=-1,高度=-1]])
	变量 根布局 : 相对布局

	事件 启动窗口:创建完毕()
		本对象.显示标题栏 = 假
	结束 事件
结束 类
```

规则：
- 页面继承 `窗口`。
- 不写 `包名`。
- 不写 `@code` 构造器样板。

## 1.1 多组件布局声明（连续块）
```t
类 测试页面 : 窗口

	@布局配置([[根布局=真,宽度=-1,高度=-1]])
	变量 根布局 : 相对布局
	@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=1,内容="标题",字体大小=24,@位于布局中间=真]])
	变量 标题文本 : 文本框
	@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=2,内容="提示文本",字体大小=18,@位于某组件之下=1,@位于布局中间=真]])
	变量 分数文本 : 文本框
	@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=3,内容="等待结果",字体大小=20,@位于某组件之下=2,@位于布局中间=真]])
	变量 结果文本 : 文本框
结束 类
```

规则：
- `类 ...` 后保留一个空行，再开始布局声明块。
- 每个 `@布局配置` 与其后 `变量` 必须紧邻。
- 各声明对之间不插空行；相邻 `@布局配置` 定义之间不允许出现空行。
- 布局声明块结束后的空行不作约束。
- 布局值支持多类型：文本（带引号）、数字、逻辑型（`真/假`），并与目标属性/方法签名匹配。
- 颜色布局值（如 `背景颜色`）必须使用十进制有符号整数，禁止 `0x/0X` 十六进制写法。
- 文本默认不引入无意空格：例如 `猜正面`、`猜反面`、`总次数:0|胜利:0|失败:0`。

## 6. 泛型容器（栈模板、哈希表等）

### 6.1 栈模板类（LIFO）
适用场景：
- 表达式求值、撤销/回退、深度遍历辅助栈。
- 只关心“最后压入、最先取出”的顺序。

建议接口：
- `压栈(成员)`：入栈
- `出栈()`：出栈并返回顶部
- `取栈顶()`：只读顶部不弹出
- `数量()`：当前元素数
- `是否为空()`：空判断

```t
类 栈模板类<模板类型1>
	方法 压栈(成员 : 模板类型1)
	结束 方法

	方法 出栈() : 模板类型1
	结束 方法

	方法 取栈顶() : 模板类型1
	结束 方法

	方法 数量() : 整数
	结束 方法

	方法 是否为空() : 逻辑型
	结束 方法
结束 类
```

### 6.2 哈希表模板类（Key-Value）
适用场景：
- 按键快速查值（配置表、缓存、索引映射）。
- 需要“键唯一、值可更新”的容器语义。

建议接口：
- `置项目(键, 值)`：新增或覆盖
- `取项目(键)`：按键取值
- `含键(键)`：键存在性判断
- `移除项目(键)`：删除键值对
- `数量()`：键值对数量

```t
类 哈希表模板类<模板类型1,模板类型2>
	方法 置项目(键 : 模板类型1, 值 : 模板类型2)
	结束 方法

	方法 取项目(键 : 模板类型1) : 模板类型2
	结束 方法

	方法 含键(键 : 模板类型1) : 逻辑型
	结束 方法

	方法 移除项目(键 : 模板类型1)
	结束 方法

	方法 数量() : 整数
	结束 方法
结束 类
```

### 6.3 队列模板类（FIFO）
适用场景：
- 任务排队、消息队列、广度遍历。
- 关注“先进入、先处理”的顺序语义。

建议接口：
- `入队(成员)`
- `出队() : 模板类型1`
- `取队首() : 模板类型1`
- `数量() : 整数`
- `是否为空() : 逻辑型`

### 6.4 选择建议（快速决策）
- 需要后进先出：选栈模板类。
- 需要按键快速定位：选哈希表模板类。
- 需要先进先出：选队列模板类。
- 生成时优先复用项目中已有模板签名，避免发明新接口命名。

## 10. 外部 Java 组件封装（导入 + 视图覆写）
```t
@外部Java文件("./TimeTextView.java")
@导入Java("rn_1.TimeTextView")
类 时间显示框 : 文本框
	@code
	public #<时间显示框>(android.content.Context context) {
		super(context);
	}

	@Override
	public TimeTextView onCreateView(android.content.Context context) {
		TimeTextView view = new TimeTextView(context);
		return view;
	}

	@Override
	public TimeTextView getView() {
		return (TimeTextView) view;
	}
	@end
结束 类
```

规则：
- `@导入Java` 用全限定名（如 `rn_1.TimeTextView`）声明导入。
- 导入后，在 `@code` 里统一使用简单类型名（`TimeTextView`）做返回类型、局部变量和强转。
- 不要在 `onCreateView/getView` 里混用 `rn_1.TimeTextView` 这类包前缀类型名。
