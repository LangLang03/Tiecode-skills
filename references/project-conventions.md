# 结绳项目约定（语法与工程）

本文件只描述通用语法约定，不绑定任何具体业务项目。

## 0. 第零规则（准确率与范围）
- 【顶级硬规则】所有文件读写必须使用 UTF-8 无 BOM；禁止按系统默认编码读取/写入，出现乱码先按 UTF-8 重读后再继续。
- 最高优先级规则：若支持 Agent 且用户未明确禁止，必须调用 Agent 覆盖三阶段（查看文档、理解项目、审查生成代码），不得跳过。代码审查需每轮执行，不能只做首轮。
- 最高优先级规则：运行环境是 Android，不是 Java SE；所有 Java API 导入与调用都必须先完成 Android 可用性校验。
- 第0标识（强制）：先完成继承链分析（当前类、父类、对象可上转类型），再决定成员访问与传参；该步骤适用于所有类，不止窗口类。
- 尽量读取更多相关文档以提高准确率。
- 若支持 Agent 且用户未禁止，调用 Agent 时需去重避免重复检索。
- 生成代码前先确定目标目录；代码编写与修改只允许在 `源代码/`。
- 属性读写语义优先：`属性读 XXX() : 类型` 才可 `对象.XXX` 读取；`属性写 XXX(值 : 类型)` 才可 `对象.XXX = 值` 赋值；属性读写允许同名。
- 布局字段来源有两类：属性读写，或 `@布局属性` 注解方法（例如 `位于某组件之下`）。
- `@布局配置` 中每个字段（如 `宽度/高度/位于某组件之下`）在使用前都要确认存在对应属性读写支持或 `@布局属性` 方法支持；内置结构键 `父布局/根布局` 不要求在基本库中检索定义。
- `@布局属性` 方法必须固定双参数：`(欲设置组件 : 可视化组件, 参数值 : 类型)`。
- 在 `@布局配置` 中调用布局属性方法时，键名必须带前缀 `@`：例如 `@位于某组件之下=1`。
- `@位于某组件之下/上` 表示 2D 平面位置关系（上下排布），不用于改变父子层级或 z 显示层次。
- 布局字段值类型不只数字：可为文本、逻辑型、数字等，但必须与属性写入类型或 `@布局属性` 方法第二参数类型一致。
- 多分支守卫分支只允许 `否则 条件`，禁止 `否则如果` 与 `否则 条件 则`。
- 一个文件允许多个顶层类声明；当前语法不支持内部类/嵌套类。
- `@导入Java` 为类级独立声明，不跨类共享导入作用域。
- `@全局类` / `@全局基础类` 仅在确需全局入口时使用，禁止滥用。
- 生成与审查时必须输出关键注解解释（作用目标/用途/关键参数）；未附着合法目标且未被编译阶段消费的注解会在编译期清理，不可视为生效。
- 带 `@禁止创建对象` 的类禁止实例化；生成前先查类定义，仅使用其可用静态成员。
- `变量 名称 : 类型?` 语义为“只声明不创建”；可用于延迟赋值与禁止创建对象场景。
- 标识符禁止使用语法关键字（类/方法/变量/参数/常量/事件命名均适用）。
- 数组类型使用后缀 `[]`，并支持多维 `[][][]`。
- 数组允许花括号初始化：`变量 名称 : 整数[] = {1,2,3}`（多维用嵌套花括号）。
- `小数` 不支持直接转 `单精小数`；仅在必须转换时，使用 `到文本().到单精小数()`（或项目中等价已定义方法）或 `@code`。

## 1. 目录约定
- `源代码/`：主业务 `.t` 源码目录。
- `绳包/`：可复用组件与依赖集合目录（库封装、资源、扩展组件等）。
- `素材/`（可选）：图片、字体、模板、静态资源等。

硬规则：
- 业务代码只写在 `源代码/`。
- `绳包/` 作为依赖目录处理，不在其任何位置写代码。

## 2. 注解引入依赖与资源

常见方式：

```t
@导入Java("android.view.View")
@外部依赖库("../../依赖库/xxx.aar")
@外部依赖库("../../依赖库/xxx.jar")
@外部Java文件("extra_java/xxx.java")
@附加资源("../../素材")
```

规则：
- 仅在使用内嵌 Java 时才添加 `@导入Java`。
- `@导入Java` 与 `@code` 中使用的 Java API 必须先验证 Android 可用性；不得直接使用 Java SE 专属 API（如 `java.awt.*`、`javax.swing.*`、`java.applet.*`）。
- 用户未明确指定 AndroidX 时，禁止引入 AndroidX（`androidx.*`）依赖与 API。
- 同文件多类场景下，`@导入Java` 需按类独立声明，不能把前一个类的导入当作后一个类的默认导入。
- 外部 Java 组件封装场景（`@外部Java文件` + `@导入Java("包名.类型名")`）下，`@code` 中 `onCreateView/getView` 的返回类型、局部变量、强转统一使用简单类型名 `类型名`，不要混用 `包名.类型名`。
- 外部库路径统一用相对路径，优先放在 `依赖库/` 或 `绳包` 内。
- 资源目录通过 `@附加资源` 暴露给项目。
- `so`、`jar`、`aar` 均按“外部依赖库”模式管理，路径保持可追踪。
- 权限与清单注解按语法用途区分：
  - `@附加权限(...)`：权限声明
- `@附加清单(...)` / `@附加清单.全局属性(...)` / `@附加清单.组件属性(...)`：清单片段声明
- `@附加可变清单([[...${参数名}...]])` + `@编译时处理参数`：编译期模板注入
- `@附加可变清单` 场景必须使用“单独类 + 单个空方法”补全占位参数；方法仅用于参数映射（例如 `方法 XXX(输入法类名:文本,输入法名称:文本,输入法配置:文本) ...`），不得混入运行时逻辑

组件封装示例（正确）：
```t
@外部Java文件("./TimeTextView.java")
@导入Java("rn_1.TimeTextView")
类 时间显示框 : 文本框
	@code
	@Override
	public TimeTextView onCreateView(android.content.Context context) {
		TimeTextView view = new TimeTextView(context);
		return view;
	}

	@Override
	public TimeTextView getView() {
		return (TimeTextView) view;
	}
	@end
结束 类
```

## 3. 布局模式（注解驱动）

`@布局配置` 是高频布局声明模式，常见写法：

```t
@布局配置([[根布局=真,宽度=-1,高度=-1]])
变量 相对布局1 : 相对布局
@布局配置([[父布局=相对布局1,宽度=-1,高度=-1,ID=1]])
变量 卡片布局1 : 卡片布局
```

规则：
- 先声明父布局，再声明子布局。
- `父布局=xxx` 明确层级关系。
- 组件属性（宽高、边距、对齐、背景）放在同一布局注解块。
- `@布局配置` 每个键都必须可映射到组件可用的属性读/属性写，或可映射到带 `@布局属性` 注解的方法；检索不到就不得使用。内置结构键 `父布局/根布局` 不受此检索约束。
- `宽度/高度/边距/对齐/背景` 等通常按属性字段处理；`位于某组件之下` 等按布局属性方法字段处理；`父布局/根布局` 视为布局系统内置结构键。
- `@布局属性` 方法字段在 `@布局配置` 中必须使用 `@方法名=值` 形式，禁止写成 `方法名=值`。
- `@位于某组件之下/上` 只改变平面排布关系，不改变组件层次关系与 z 顺序。
- 多组件布局声明块格式：类声明后先空一行；随后每个 `@布局配置` 与其后 `变量` 之间不可插空行，各组之间不可插空行；`@布局配置` 定义之间禁止出现空行。
- 布局声明块结束后不再要求“是否空行”。
- `内容`、`字体大小` 等可在布局块中直接赋值（例如 `内容="猜硬币游戏"`,`字体大小=24`），前提是目标组件存在对应属性写入支持。
- 颜色类布局值（如 `背景颜色`）必须写十进制有符号整数，禁止 `0x/0X` 十六进制字面量（例如 `背景颜色=-657931`，不要写 `背景颜色=0xfff5f5f5`）。
- 普通对象变量声明 `变量 名称 : 类型` 自动创建对象；禁止写 `变量 名称 : 类型 = 创建 类型()`，`创建` 仅内部使用。
- 带 `@禁止创建对象` 的类型禁止自动创建或显式创建；优先用 `类型.静态方法(...)`，如需占位引用使用 `变量 名称 : 类型?`。
- 多分支条件统一写法为 `否则 条件`，禁止 `否则如果`。
- 多分支条件中 `否则 条件` 后禁止再写 `则`。
- 条件循环统一写法为 `循环(条件)`，禁止 `循环 条件`。
- 循环提前退出统一使用 `退出循环`；结绳层禁止使用 Java `break`（`break` 仅允许在 `@code`）。
- 函数调用需严格匹配已定义签名参数，不可自行发明重载调用形式；同时必须匹配方法类别：实例方法写 `对象.方法(...)`，静态方法写 `类型.方法(...)`。
- 数组声明统一使用后缀维度语法：`类型[]`、`类型[][]`、`类型[][][]`。
- 数组初始化统一使用花括号字面量：`变量 名称 : 类型[] = {值1,值2,值3}`。
- `小数 -> 单精小数` 转换统一要求“非直接转换”：只在必须转换时走 `到文本().到单精小数()`（或项目中等价已定义方法）或 `@code` 路径。

错误示例（禁止）：
```t
@布局配置([[父布局=根布局,宽度=-2,height=-2,内容="猜 正面"]])
变量 正面按钮 : 按钮
```

修正示例：
```t
@布局配置([[父布局=根布局,宽度=-2,高度=-2,内容="猜正面"]])
变量 正面按钮 : 按钮
```

错误示例（禁止：十六进制颜色 + 定义间空行）：
```t
@布局配置([[根布局=真,宽度=-1,高度=-1,背景颜色=0xfff5f5f5]])
变量 根布局 : 相对布局

@布局配置([[父布局=根布局,宽度=-1,高度=200,ID=1,背景颜色=0xff2c3e50,@位于布局顶部=真]])
变量 显示区域 : 相对布局
```

修正示例（十进制颜色 + 连续定义无空行）：
```t
@布局配置([[根布局=真,宽度=-1,高度=-1,背景颜色=-657931]])
变量 根布局 : 相对布局
@布局配置([[父布局=根布局,宽度=-1,高度=200,ID=1,背景颜色=-13877680,@位于布局顶部=真]])
变量 显示区域 : 相对布局
```

示例（正确）：
```t
@布局配置([[根布局=真,宽度=-1,高度=-1]])
变量 根布局 : 相对布局
@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=1]])
变量 标题文本 : 文本框
@布局配置([[父布局=根布局,宽度=-2,高度=-2,ID=2,内容="提示文本",字体大小=18,@位于某组件之下=1,@位于布局中间=真]])
变量 分数文本 : 文本框
```

## 4. 窗口与页面定义规则
- 主窗口固定写法：`类 启动窗口 : 窗口`。
- 页面类统一继承 `窗口`。
- 不把页面类生成为 `组件容器`。
- 默认不写 `包名 ...`。
- 构造器样板采用“先检查后补充”：若类中已定义则不得重复生成；仅在缺失且必要时补充一次。

页面骨架示例：
```t
类 启动窗口 : 窗口
	@布局配置([[根布局=真,宽度=-1,高度=-1]])
	变量 根布局 : 相对布局

	事件 启动窗口:创建完毕()
		本对象.显示标题栏 = 假
	结束 事件
结束 类
```

## 5. 本对象 / 父对象 语义
- `本对象`：当前对象，语义等价 Java `this`。
- `父对象`：父类对象语义，使用方式与 Java `super` 一致。
- 父对象/子对象的访问与调用链按 Java 继承规则理解。
- 继承链分析为强制步骤：先确认当前类继承关系，再决定是否可直接把对象当作父类型传参。
- 当类继承 `窗口`（含间接继承）时，`本对象` 本身就是 `安卓环境`（Context）；需要 `安卓环境` 的函数应直接传 `本对象`。
- 在继承 `窗口` 的类中，禁止生成 `本对象.取安卓环境()`。
- `取安卓环境()` 仅用于“当前对象不是 `安卓环境` 且确实需要转取环境”的场景。

示例：
```t
事件 页面:创建完毕()
	本对象.显示标题栏 = 假
结束 事件
```

传参示例（继承 `窗口` 时）：
```t
变量 打开成功 : 逻辑型 = 打开应用(本对象, "com.demo.app")
```

## 6. 事件优先的页面逻辑
- 页面生命周期、组件回调优先使用 `事件` 块实现。
- 纯工具逻辑与复用能力优先用 `方法`。

示例：
```t
事件 页面:创建完毕()
	订阅事件()
结束 事件
```

## 7. 线程与回调模式
- 耗时逻辑放 `提交到新线程运行()`。
- UI 变更放主线程块。
- 网络回调通常采用“请求对象 + 事件 请求成功/请求异常”模式。

## 8. `@code` 使用边界
- 构造器样板是可用模式，但必须先检查是否已定义：
```t
@code
public #<测试页面>(android.content.Context context) {
	super(context);
}
@end
```
- 若上述构造器已存在，不得重复生成。
- 只有在结绳语句无法表达时才使用 `@code`。
- 可用结绳语句表达的逻辑，全部留在结绳层实现。

## 9. 函数调用校验（存在性 + 签名 + 静态/实例形态）
- 在调用任意函数前，先在 `源代码/` 与 `绳包/` 搜索其定义或可见声明。
- 搜索不到即视为不可用，不得假设其存在。
- 调用前必须确认方法类别：若为实例方法，必须由实例接收者调用（`对象.方法(...)`）；若为静态方法，必须由类型接收者调用（`类型.方法(...)`）。
- 若类带 `@禁止创建对象`，调用前必须确认该类无实例化路径，仅按静态成员方式使用。
- 禁止把实例方法写成全局调用（例如禁止 `到文本(变量名)`，应写 `变量名.到文本()`）。
- 不存在“未公开内置函数”；任何可调用能力都应可在代码或文档中找到依据。

## 10. 生成代码时的硬约束
- 块闭合完整：`结束 类/方法/属性/事件/循环/如果/假如`。
- `结束 类` 只闭合当前类；文件可连续声明多个顶层类，不得声明内部类。
- 注解靠近声明，不要放在方法体内部（`@code` 中的 Java 注解除外）。
- 布局链路必须保证父组件先定义。
- 依赖与资源路径可解析，避免写死绝对路径。
- 条件表达式使用比较符号：相等/不等使用 `==` / `!=`；`=` 只用于赋值与默认值，禁止写出 `如果 硬币结果 = 0 则`。
- 禁止使用关键字命名标识符；发现冲突时必须重命名。

- 结绳布局底层是 Java：布局设计/实现可按 Java 思路落地，但必须先确认有对应实现类；若缺失则通过 `@code` 封装实现并保持可追踪。
