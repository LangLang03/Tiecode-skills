# 控制流、容错与并发语法

## 1. 条件
```t
如果 条件 则
	// 分支A
否则
	// 分支B
结束 如果
```

支持多分支：
```t
如果 条件1 则
	...
否则 条件2
	...
否则
	...
结束 如果
```

分支写法规定（强制）：
- 使用 `否则 条件` 作为第二分支写法，禁止 `否则如果`。
- `否则 条件` 行末不允许再写 `则`；`否则 条件 则` 属于非法写法。
- 错误示例：`否则 条件2 则`
- 正确示例：`否则 条件2`

逻辑符号规定（强制）：
- 条件判断中的“相等/不等比较”使用 `==` / `!=`。
- `=` 仅用于赋值和默认值，不可作为条件比较符。
- 错误示例：`如果 硬币结果 = 0 则`
- 正确示例：`如果 硬币结果 == 0 则`

## 2. 假如/是 分支
```t
假如 状态
是 0
	返回 "初始"
是 1
	返回 "运行"
结束 假如
```

## 3. 循环

计数循环：
```t
循环(i,0,取数组长度(数组))
	处理(数组[i])
结束 循环
```

条件循环：
```t
循环(长度 != -1)
	长度 = 输入流.读取(缓冲)
结束 循环
```

条件循环括号规则（强制）：
- 条件循环必须写成 `循环(条件)`。
- 错误示例：`循环 长度 != -1`
- 正确示例：`循环(长度 != -1)`

遍历语法：
```t
循环(集合 -> 值)
	处理(值)
结束 循环
```

退出循环关键字：
```t
循环(真)
	如果 命中退出条件 则
		退出循环
	结束 如果
结束 循环
```

`退出循环` 规则（强制）：
- 仅可写在 `循环 ... 结束 循环` 代码块内部。
- 结绳层退出循环必须使用 `退出循环`，不得写 Java `break`。
- Java `break` 仅允许出现在 `@code` 代码块中。

## 4. 容错处理

块式容错：
```t
容错处理()
	可能异常调用()
结束容错()
```

与返回结合：
```t
方法 打开应用(...) : 逻辑型
	容错处理()
	返回 真
	结束容错()
	返回 假
结束 方法
```

## 5. 线程与异步

异步标签：
```t
@异步方法
方法 加载网络图片(网址 : 文本)
	变量 字节集 = 等待 网络.取网页源码_字节集_同步(网址)
结束 方法
```

线程块：
```t
提交到新线程运行()
	延时(100)
	提交到主线程运行(本对象)
		刷新界面()
	结束提交到主线程()
结束提交到新线程()
```

## 6. 控制流生成约束
- `结束 xxx` 必须匹配最近未闭合块。
- 在 `@code` 中写 Java 控制流；在 结绳层写 `如果/循环/假如`。
- 不把 Java `if/for` 与 结绳 `结束 如果/循环` 混搭。
- 条件比较中禁止使用赋值符 `=`；请使用 `==` / `!=`。
- 循环提前退出请使用 `退出循环`；禁止在 结绳层使用 `break`。
